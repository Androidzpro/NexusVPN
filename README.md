## 📡 UDP Custom (rango 1–65535)

NexusVPN Pro v4.0 implementa un sistema completo de **UDP Custom con soporte de rango total**:

```bash
# Configurar durante instalación:
  Rango UDP (ej: 10000-65000 o '1-65535' para todos): 1-65535

# O desde el menú (opción 5 → 7):
  Puerto inicial: 10000
  Puerto final  : 65000
  ✔  Rango UDP 10000-65000 abierto
```

### ¿Cómo funciona UDP Custom?

```
 Cliente (app)  ──UDP──▶  Puerto cualquiera (10000-65535)
                                    │
                          socat/redirección
                                    │
                            127.0.0.1:7300 (BadVPN)
                                    │
                          SSH / Xray (tunnel final)
```

Se instala el script `/usr/local/bin/udp-custom` y se crean servicios systemd individuales por puerto, todos con `Restart=always`.

### Backup automático diario

El instalador configura un **cron job a las 3:00 AM** que crea un backup automático cada día. Se conservan los **últimos 10 backups**; los más antiguos se eliminan automáticamente.

```bash
# Crontab generada automáticamente:
0  3 * * *  /usr/local/bin/nexusvpn --backup
0  * * * *  /usr/local/bin/nexusvpn --clean-keys
0  4 * * 0  /usr/local/bin/nexusvpn --restart-services  # (opcional)
```

## 🛡️ Seguridad Avanzada

### Autenticación del panel
- Contraseña de admin con hash **SHA-512**
- Máximo **3 intentos** fallidos → cierre automático de sesión
- Registro completo de todos los accesos en `/var/log/nexusvpn/access.log`
- Cada sesión queda registrada con timestamp y resultado

### Protección de datos
- Keys almacenadas con hash **SHA-256**, nunca en texto plano
- Solo los primeros 8 caracteres de cada key son visibles en pantalla
- Base de datos con permisos `chmod 600` (solo root)
- Token del bot con permisos `chmod 600`

### Protección de red
- **Fail2ban** activo contra ataques SSH brute force
- **Rate limiting** en UFW para el puerto SSH
- Detección y bloqueo de IPs desde el bot y el panel web
- Logs de Xray con registro de emails de usuarios

### Antes de poner en producción

```bash
# ✅ OBLIGATORIO — Cambiar contraseña admin:
nexusvpn  # → Opción 14

# ✅ RECOMENDADO — Activar licencia:
nexusvpn  # → Opción 1 → 1

# ✅ RECOMENDADO — Configurar bot de Telegram:
nexusvpn --bot-token <TU_TOKEN>

# ✅ OPCIONAL — Instalar SSL con dominio:
```

## ❓ FAQ

<details>
<summary><b>¿Cuál es la diferencia entre v3.0 y v4.0?</b></summary>

La v4.0 es una reescritura completa con 5,247 líneas (vs 2,567 de v3.0). Agrega WireGuard, AmneziaWG, IKEv2, bot de Telegram, panel web, sistema de logs avanzado, monitoreo con países, UDP Custom completo (1-65535), BadVPN con puertos dinámicos y verificación del sistema antes de instalar.

</details>

<details>
<summary><b>¿Funciona en el Free Tier de Oracle Cloud (ARM)?</b></summary>

Sí. El instalador detecta automáticamente la arquitectura `aarch64` y descarga los binarios correctos de Xray, Hysteria2 y WireGuard para ARM64. Oracle Cloud Free Tier (VM.Standard.A1) es totalmente compatible.

</details>

<details>
<summary><b>¿Puedo instalar el bot de Telegram después de la instalación principal?</b></summary>

Sí. Desde el menú principal → opción 24 → instalar bot. O bien:
```bash
nexusvpn --bot-token <TU_TOKEN_DE_BOTFATHER>
```

</details>

<details>
<summary><b>¿El panel web es seguro?</b></summary>

El panel web usa autenticación con contraseña. Por seguridad adicional, se recomienda: (1) cambiar el puerto por defecto `8080`, (2) poner Cloudflare frente al panel, (3) agregar autenticación básica en Nginx. El panel web escucha solo en `127.0.0.1:5000` y se expone vía Nginx reverse proxy.

</details>

<details>
<summary><b>¿IKEv2 funciona con iPhone sin instalar app?</b></summary>

Sí. IKEv2 es compatible de forma nativa con iOS/iPadOS en Ajustes → General → VPN y gestión de dispositivos → VPN. Solo necesitas importar el certificado `.p12` generado automáticamente y configurar los datos del servidor.

</details>

<details>
<summary><b>¿Puedo migrar desde v3.0 a v4.0?</b></summary>

```bash
# 1. Haz backup con v3.0:
nexusvpn  # → Opción 13 → 1

# 2. Instala v4.0:
bash install.sh --install

# 3. Restaura desde backup (opción 11 → 2)
# Los usuarios y keys de v3.0 son compatibles.
```

</details>

<details>
<summary><b>¿Cómo cambiar el puerto de un protocolo sin reinstalar?</b></summary>

```bash
nexusvpn  # → Opción 10 → seleccionar protocolo → nuevo puerto

# El servicio se reinicia automáticamente y UFW se actualiza.
# No se pierde ninguna configuración de usuarios.
```

</details>

<details>
<summary><b>¿Cómo desinstalar completamente?</b></summary>

```bash
bash <(wget -qO- https://raw.githubusercontent.com/Androidzpro/NexusVPN/main/uninstall.sh)

```
